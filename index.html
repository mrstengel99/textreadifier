Tone slides between subdued (quieter, a touch lower — like Hushed) and emphatic (louder, a touch brighter — like Emphatic) — it's the same combination those presets use, as a continuous dial. Volume alone barely registers on most voices, so Tone nudges pitch along with it to keep the effect audible.

All three sliders apply immediately to every passage you haven't manually customized — so if the whole chapter reads too fast, slow it down here rather than passage by passage. New chapters start at these settings too.

Cloud voices — for the final MP3

These 18 names are Microsoft's free cloud voices — what your exported script actually uses to render the real MP3 in the Python step. Auditioning here plays an approximation, built by guessing which of your installed voices (above) sounds closest — it is not the real cloud voice itself, since a browser page can't reach that service directly. If a name above matches one in the "Voices on your computer" list exactly (like Jenny, Aria, or Guy on Windows 11), the approximation becomes the real thing; otherwise treat it as a rough guide only.

Record chapter — real MP3, no API key

Records whatever's actually playing (the free preview voices above, or real generated audio if that's turned on) as it reads the whole chapter, then saves it as a real MP3 — no API key, no Python. Chrome only. Clicking this opens Chrome's own "Share Tab" picker — choose This Tab and make sure Share tab audio is checked, then click Share. Recording takes as long as the chapter takes to read out loud (a few minutes is normal) — keep this tab open and don't mute it while it runs.
